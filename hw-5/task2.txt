1. ЛОГИКА POST CONTROLLER

класс PostsController нужен для контроля постов. Во всех экземпляров этого класса инициализируется переменная экземпляра @posts - пустой массив.

также класс расширен модулем Resource и имеет его функционал (делается это с помощью extend!)

далее в коде прописаны функции, которые нужны для взаимодействия с постами



2. КЛАСС ROUTER

Во всех экземпляров этого класса также инициализируется переменная экземпляра, но только  @router - пустой хэш.

функция init содержит в себе метод resources, который создает маршруты для методов в PostsController

далее используется цикл, который лупит надпись до тех пор, пока не введется "q" (после выводится надпись Good bye! - цикл завершается)

если юзер выбрал 1, то метод connection у PostsController возвращает соединение, в настоящее время связанное с классом.
